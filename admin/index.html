Admin Guide (Quick)

This panel lets you edit site data stored in simple YAML files. Items on the left are the files and their entries; the right side is the form to edit the selected item. Saving makes a GitHub commit and the site updates automatically.

Make a quick edit
  1. Click Login and paste a GitHub personal access token with repo access.
  2. On the left:
    • Pick a _data/*.yml file from the dropdown.
    • Use Search to find an entry; click to open it.
    • Add, Duplicate, or Delete when needed.
  3. On the right:
    • Edit fields (text, dates, toggles, lists). For lists, use ＋ Add to add rows.
    • Reorder items with Move Up/Down.
  4. Click 💾 Save. The site redeploys automatically. If you don’t see changes, hard-refresh (Ctrl/Cmd+Shift+R).
File types
  • Lists: left shows all items; right edits the selected one.
  • Maps (key → value): you’ll see a 2-column table with Add Row.
Tips
  • ↻ Reload pulls the latest from GitHub; ⬇ Export downloads current YAML.
  • “Display key” controls the label used for each item in the list.

Admin Handbook — Full Guide & Troubleshooting

This handbook covers content edits, assets, database, DNS, OTP/email, and what to do if the Super Admin is unavailable.

A) Access & Credentials
  • Email: manilacorinthianchapter@gmail.com
  • Password: ask Super Admin or reset via Gmail if you have inbox access.
  • 2FA: request code from Super Admin.
B) Editing Website Content
  1. Open /admin.
  2. Create/use a fine-grained GitHub token for repo manilacorinthianchapter/mcc (Contents: Read/Write).
  3. Paste token into the admin UI → edit → Save/Publish (commits to GitHub Pages).
  4. Troubleshoot: 401 → wrong token scope; site not updating → check GitHub Actions/Pages logs; hard-refresh.
C) Assets (Cloudflare R2)
  1. Cloudflare → R2 → assets/ (organize by folders).
  2. Public URLs: asset…/list
  3. Troubleshoot: 404/403 → check path/case & Worker; old image → hard-refresh/purge cache.
D) Database (Supabase)
  1. Open Supabase.
  2. Backup (CSV) before edits. Respect RLS and RPCs (verify_password, etc.).
  3. Troubleshoot: writes fail → check RLS, auth session/role; missing data → column names/permissions.
E) Domain (Porkbun)
  1. Porkbun → Domain → DNS. Keep GitHub Pages + subdomain records correct.
  2. Propagation can take time; verify and wait.
F) OTP / Emails (Resend)
  1. Resend: domain verification, SPF/DKIM, API keys.
  2. OTP not arriving → check logs/bounces; rotate key if compromised and update the site.
G) If the Super Admin is Unavailable
  1. Use Gmail recovery to reset provider passwords (GitHub/Cloudflare/etc.) via inbox control.
  2. Document all changes (who/what/when) for later reconciliation.
H) Quick Checks
  • Admin button hidden? In console: 
    localStorage.getItem('u_role')  // "2" or "3"
sessionStorage.getItem('gh_token') // if you store GitHub token
  • Assets missing? Verify in list, match case, purge cache.
  • DB issues? Check policy/role/session; confirm expected columns.
  • Site not updating? Check GitHub Pages/Actions build logs.

Keep tokens/keys secure. Revoke ones you no longer use.
